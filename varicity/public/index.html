Visualization

Visualization interactions

Rotating the camera: left click + drag
Moving the camera: right click + drag
Zooming in / out: mouse wheel

Legend

Buildings

Colors

  • Yellow buildings represent VPs being hotspots
  • Blue buildings represent variants being hotspots
  • Grey buildings represent all classes (including VPs and variants) not being hotspots

Crowns

  • Chimneys: Factory class
  • Dome: Strategy class
  • Inverted pyramid: Template class
  • Sphere: Decorator class
  • Pyramid: Class defined as an entrypoint to build the visualization

Links

Colors

  • The red street groups classes defined as entrypoints
  • Green streets, both on the plan and underground, represent usage relationships
  • Aerial links represent inheritance relationships. They can be blue (for a class / subclass relationship) or red (for an interface / implementation relationship).
    The brightest end of the link indicates the parent. On the example above, the class on the right is a superclass.

Sidebar

  • name: Qualified name of the class (class name + package)
  • nbMethodVariants: Number of methods overloads
  • nbConstructorVariants: Number of constructors overloads
  • nbVariants: Number of class level variants (subclasses of the class / classes implementing the interface). Note: all the variants may not appear on the visualization.
